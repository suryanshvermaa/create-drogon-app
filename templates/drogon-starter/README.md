# 🚀 drogon-starter

A modern starter template for building high-performance web applications and APIs with the **Drogon C++ Web Framework** ⚡

Generated by **create-drogon-app** 📦

## ⚡ Setup (local)

1) Create local env + config files:

```bash
cp .env.example .env
cp config/config.example.json config/config.json
```

The server reads `PORT` from `.env` (defaults to `3000` if missing).

2) Start PostgreSQL (optional but recommended):

```bash
docker compose up -d
```

PostgreSQL runs on `localhost:5432` and executes any `sql/*.sql` scripts on first boot.

3) Install dependencies (Conan):

```bash
conan profile detect --force
```
```
conan install . --output-folder=build --build=missing -s build_type=Release
# or if using ubuntu or any linux distro
conan install . \
    --output-folder=build \
    --build=missing \
    -c tools.system.package_manager:mode=install \
    -c tools.system.package_manager:sudo=True
```

4) Configure + build (CMake presets):

```bash
cmake --preset conan-release
cmake --build --preset conan-release
```

## ✅ Prerequisites

- C++17 toolchain
- CMake >= 3.15
- Python 3 + pip
- Conan
- Docker (optional, used for PostgreSQL and/or building the image)

```bash
pip install conan
```

## ➕ Add a new controller

This codebase registers controllers via `HttpController` + `METHOD_ADD`, and compilation is driven by explicit source listing in `CMakeLists.txt`.

### 1) Create the controller

Create `src/controllers/ping.h`:

```cpp
#pragma once

#include <drogon/HttpController.h>

class ping : public drogon::HttpController<ping>
{
    public:
        METHOD_LIST_BEGIN
        // With HttpController, the base path is "/ping" (the class name)
        // Empty method path "" becomes: GET /ping
        METHOD_ADD(ping::get, "", drogon::Get);
        METHOD_LIST_END

        void get(const drogon::HttpRequestPtr &req,
                         std::function<void(const drogon::HttpResponsePtr &)> &&callback);
};
```

Create `src/controllers/ping.cc`:

```cpp
#include "ping.h"
#include "../utils/response.hpp"

void ping::get(const drogon::HttpRequestPtr &,
                             std::function<void(const drogon::HttpResponsePtr &)> &&callback)
{
        callback(Response::success(drogon::k200OK, "pong"));
}
```

### 2) Add it to the build

Open `CMakeLists.txt` and add the new `.cc` file to `add_executable(Drogon_Starter ...)`:

```cmake
./src/controllers/ping.cc
```

### 3) Build + test

```bash
cmake --build --preset conan-release
curl -s http://localhost:3000/ping
```

## 🗃️ Database configuration

At startup, the app loads `./config/config.json`.

- Template: `config/config.example.json`
- Local file: `config/config.json` (ignored)

If you run the app in a container and want it to talk to the Compose Postgres container, set `host` in the config to `db` (the Compose service name) instead of `127.0.0.1`.

## 🧬 ORM model generation (optional)

Models in `src/models/` are generated by `drogon_ctl` (see header comments in `Users.h/.cc`).

Template file:

```bash
cp src/models/model.example.json src/models/model.json
```

Then (if `drogon_ctl` is available on your machine):

```bash
drogon_ctl create model src/models/model.json
```

## 🐳 Docker

Build the image:

```bash
docker build -t drogon-starter .
```

Run it:

```bash
docker run --rm -p 3000:3000 drogon-starter
```
Note: if you run the app inside a Docker container and connect to the Compose DB, the DB host in the config typically needs to be `db` (service name) instead of `127.0.0.1`.
