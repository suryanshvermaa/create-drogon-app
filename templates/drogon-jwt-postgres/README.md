# 🔐 drogon-jwt-postgres

A **Drogon (C++)** backend template with:

- 🐘 PostgreSQL (Docker Compose)
- 🔑 JWT authentication (Auth filter + token utils)

Generated by **create-drogon-app** 📦

## ⚡ Setup (quick start)

```bash
docker compose up -d
cp .env.example .env

conan profile detect --force
conan install . \
  --output-folder=build \
  --build=missing \
  -c tools.system.package_manager:mode=install \
  -c tools.system.package_manager:sudo=True

cmake --preset conan-release
cmake --build --preset conan-release

./build/drogon_jwt_postgres
```

## ✅ Prerequisites

- 🧰 C++ toolchain (gcc/clang)
- 🏗️ CMake (>= 3.15)
- 📦 Conan 2.x
- 🐳 Docker (recommended for local PostgreSQL + MinIO)

## 🐳 Local services (Postgres)

Start dependencies:

```bash
docker compose up -d
```

This uses `docker-compose.yml` and provisions:

- 🐘 PostgreSQL: `localhost:5432` (db `drogon_jwt_postgres`, user `postgres`, pass `postgres`)

DB schema initialization:

- 🗃️ SQL files in `sql/` are mounted into `/docker-entrypoint-initdb.d`

## 🔧 Environment variables

The server loads environment variables from a local `.env` file at startup.

```bash
cp .env.example .env
```

From `.env.example`:

- `PORT` (defaults to `3000`)
- `JWT_SECRET`

## 🏗️ Build (local)

conan profile detect --force

## ▶️ Run (local)

Important: the server loads `./config/config.json` using a relative path, so run it from the **project root**.

```bash
./build/drogon_jwt_postgres
```

## 🐳 Container (optional)

Production compose builds/runs the backend container:

```bash
docker compose -f docker-compose.prod.yml up --build
```

Notes:

- `docker-compose.prod.yml` maps host `3000:3000`, so set `PORT=3000` in `.env` for the container.
- The `Dockerfile` declares `EXPOSE 8080`, but the app listens on `PORT` (default `3000`).
